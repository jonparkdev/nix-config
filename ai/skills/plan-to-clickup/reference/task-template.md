# Task Template

Use this template for ClickUp subtask descriptions. Convert to markdown when creating tasks via MCP.

```
**Task Title:** [Clear, Descriptive Title]

**Description:** A summary of what needs to be accomplished. Include key details that define the task and its purpose.

**Priority:** [Low/Medium/High/Urgent]

**Acceptance Criteria:**

- **Criterion 1:** Detail what constitutes success for this task. Include specific outcomes or conditions that must be met.
- **Criterion 2:** Additional requirements or functionalities that need to be addressed.
- **Criterion 3:** Standards or performance metrics that will be used to evaluate task completion.

**Resources Needed:** List any tools, systems, or additional information required to complete the task.

**Dependencies:** Identify any other tasks, projects, or external factors that this task depends on or affects.

**Notes:** Any other relevant information that will assist in executing the task.
```

## Priority Guide

Use this when mapping decomposition priority to ClickUp priority values:

| Priority | ClickUp Value | When to Use |
|----------|---------------|-------------|
| **Low** | 4 | Minimal impact on progress. Can be postponed without consequences. Non-critical improvements, tasks awaiting decisions. |
| **High** | 2 | Directly impacts project progress. Delays cascade to subsequent tasks. Bug fixes affecting users, release milestone work. |
| **Urgent** | 1 | Requires immediate attention. Affects system stability, compliance, or critical operations. Security patches, critical bugs, legal requirements. |
